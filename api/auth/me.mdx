## Endpoint

```
GET /api/auth/me
```

Returns information about the currently authenticated user. This endpoint requires a valid `rdweb_token` cookie to be present in the request.

## Authentication

<Warning>
  This endpoint requires authentication. Include the `rdweb_token` cookie in your request.
</Warning>

The endpoint validates the JWT token from the `rdweb_token` cookie. If authentication fails, one of the following errors will be returned:

- `NO_TOKEN`: No authentication cookie present
- `TOKEN_EXPIRED`: Session has expired
- `INVALID_TOKEN`: Token is malformed or signature is invalid

## Request

This endpoint does not require query parameters or a request body. Authentication is handled via the `rdweb_token` cookie.

## Response

<ResponseField name="username" type="string">
  The user's Active Directory username
</ResponseField>

<ResponseField name="displayName" type="string">
  The user's full display name from Active Directory
</ResponseField>

<ResponseField name="email" type="string">
  The user's email address
</ResponseField>

<ResponseField name="domain" type="string">
  The Active Directory domain the user belongs to
</ResponseField>

<ResponseField name="initials" type="string">
  User's initials derived from display name (first letter of first two words)
</ResponseField>

<ResponseField name="privateMode" type="boolean">
  Whether the session was created in private mode (affects session duration)
</ResponseField>

## Example Request

```bash
curl -X GET https://your-domain.com/api/auth/me \
  -H "Cookie: rdweb_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  # OR using cookies file
curl -X GET https://your-domain.com/api/auth/me \
  -b cookies.txt
```

## Example Responses

### Success (200 OK)

```json
{
  "username": "jdoe",
  "displayName": "John Doe",
  "email": "jdoe@company.com",
  "domain": "COMPANY",
  "initials": "JD",
  "privateMode": false
}
```

### Error Responses

#### No Token (401 Unauthorized)

Returned when the `rdweb_token` cookie is missing from the request.

```json
{
  "error": "No autenticado",
  "code": "NO_TOKEN"
}
```

#### Token Expired (401 Unauthorized)

Returned when the JWT token has exceeded its expiration time.

```json
{
  "error": "Sesión expirada",
  "code": "TOKEN_EXPIRED"
}
```

#### Invalid Token (401 Unauthorized)

Returned when the JWT token is malformed or has an invalid signature.

```json
{
  "error": "Token inválido",
  "code": "INVALID_TOKEN"
}
```

## Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `NO_TOKEN` | 401 | Authentication cookie not present in request |
| `TOKEN_EXPIRED` | 401 | JWT token has expired - user needs to login again |
| `INVALID_TOKEN` | 401 | JWT token signature invalid or token malformed |

## Use Cases

### Session Validation

Check if the user's session is still valid:

```javascript
async function checkSession() {
  try {
    const response = await fetch('/api/auth/me', {
      credentials: 'include'
    });
    
    if (response.ok) {
      const user = await response.json();
      console.log('Session valid:', user);
      return user;
    } else {
      console.log('Session invalid, redirect to login');
      window.location.href = '/login';
    }
  } catch (error) {
    console.error('Failed to check session:', error);
  }
}
```

### Load User Profile

Retrieve current user information for display:

```javascript
import axios from 'axios';

axios.get('/api/auth/me', {
  withCredentials: true
})
  .then(response => {
    const user = response.data;
    // Update UI with user info
    document.getElementById('user-name').textContent = user.displayName;
    document.getElementById('user-initials').textContent = user.initials;
  })
  .catch(error => {
    if (error.response?.status === 401) {
      // Handle authentication error
      redirectToLogin();
    }
  });
```

### Periodic Session Check

Implement automatic session validation:

```javascript
// Check session every 5 minutes
setInterval(async () => {
  try {
    const response = await fetch('/api/auth/me', {
      credentials: 'include'
    });
    
    if (!response.ok) {
      // Session expired, notify user
      showSessionExpiredModal();
    }
  } catch (error) {
    console.error('Session check failed:', error);
  }
}, 5 * 60 * 1000);
```

## Implementation Notes

- User information is extracted from the JWT token payload, not queried from Active Directory
- The endpoint is fast as it doesn't require external service calls
- Initials are computed server-side using the first letter of the first two words in `displayName`
- The `privateMode` flag indicates whether the session has extended timeout (240 min vs 20 min)
- User groups are stored in the JWT but not returned by this endpoint
- Token validation happens in the `authenticate` middleware before the route handler executes
